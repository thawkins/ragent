# Structured Tool Output

### From: copy_file

Structured tool output represents a design pattern for returning rich, machine-readable results from tool executions rather than simple success/failure indicators. CopyFileTool implements this pattern through the `ToolOutput` struct, which separates human-readable presentation content from structured metadata. The content field provides a formatted string describing the operation (`Copied 'source' → 'destination' (1234 bytes)`), while the metadata field contains a JSON object with precise, queryable data fields. This dual representation supports both immediate user feedback and downstream programmatic processing, enabling logging, analytics, and further automation based on operation results.

The architectural significance of structured output becomes apparent in agent systems where tools compose into workflows. The metadata enables precise tracking of file operations: exact source and destination paths (after resolution), byte counts for transfer verification, and extensibility for additional fields like checksums or timestamps. JSON's ubiquity ensures interoperability with diverse consumers, from JavaScript frontends to Python analytics pipelines. The `Option<Value>` type for metadata allows tools to return content-only results when structured data provides no value, while the mandatory content ensures human intelligibility. This design balances the needs of autonomous systems (precise data) with human oversight (readable descriptions).

Implementation patterns for structured output vary across ecosystems. CopyFileTool's approach aligns with OpenAI's function calling and similar LLM-tool interfaces, where JSON schemas define both inputs and outputs. The serialization uses `serde_json` for efficient, type-safe JSON construction. Future extensions might incorporate streaming output for large file operations, progress indicators, or structured error responses with machine-readable error codes. The pattern also enables idempotency tracking—by returning consistent metadata for identical operations, callers can detect whether an operation was truly performed or deduplicated. These capabilities position structured output as foundational infrastructure for reliable, observable automation systems.
