# Competitive Gap Analysis and Feature Recommendations for ragent

This strategic analysis document compares ragent against four key competitors (ClaudeCode, OpenCode, GitHub Copilot CLI, and RooCode) to identify feature gaps and prioritize development recommendations. The analysis covers 12 major categories including context management, shell execution, agent modes, cost tracking, extensibility, UI/UX, performance, and security. Each gap is assessed for impact (HIGH/MEDIUM/LOW) and implementation complexity, with specific recommendations provided. The document also outlines a phased implementation roadmap spanning 10 phases from core feature parity to advanced security and polish features. Key findings highlight that ragent lags in persistent project memory, shell session persistence, MCP protocol support, and cost transparency while maintaining unique strengths in Rust performance, terminal-first design, multi-provider support, and team/swarm orchestration. Competitive positioning recommendations emphasize doubling down on these differentiators to target DevOps/SRE professionals and enterprise self-hosted deployments.
